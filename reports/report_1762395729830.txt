TYPOGRAPHY EXPERT AGENT
═══════════════════════════════════════════════════════

Viewport: mobile
Specialty: Typography & Readability

📋 FINDINGS:
───────────────────────────────────────────────────────
1. [MAJOR] Font Size Analysis
   Body text should be at least 16px for optimal readability on mobile devices.

2. [MINOR] Line Height
   Line height (leading) should be between 1.4-1.6 times the font size for comfortable reading.

3. [MAJOR] Typographic Hierarchy
   Establish clear hierarchy using font size, weight, and spacing to distinguish between headings, subheadings, and body text.

💡 RECOMMENDATIONS:
───────────────────────────────────────────────────────
1. Optimize Body Text Readability
   Priority: high | Effort: low
   Adjust font size, line height, and line length to meet readability best practices.
   Implementation:
     • Set minimum font size to 16px
     • Apply line-height: 1.6 for body text
     • Limit line length to 45-75 characters
     • Use comfortable letter-spacing

2. Enhance Typography Hierarchy
   Priority: high | Effort: medium
   Create clear visual distinction between heading levels and body text.
   Implementation:
     • Use modular scale for heading sizes (e.g., 1.25, 1.5, 2)
     • Apply font-weight to differentiate importance
     • Increase spacing above headings
     • Consider using different font for headings
     • Maintain consistent hierarchy across all pages

3. Improve Text Contrast
   Priority: critical | Effort: low
   Ensure text meets WCAG AA standards with minimum 4.5:1 contrast ratio.
   Implementation:
     • Test contrast ratios with accessibility tools
     • Darken text or lighten backgrounds where needed
     • Avoid gray text below #767676 on white
     • Consider different color for links

4. Mobile Typography Optimization
   Priority: medium | Effort: low
   Optimize typography specifically for mobile reading experience.
   Implementation:
     • Increase font size for small screens if needed
     • Reduce line length for easier reading
     • Adjust spacing for touch interfaces
     • Test readability in various lighting conditions

   Priority: high | Effort: medium

     • Add success/error feedback after form submissions

   Ensure all text meets WCAG AA standards (AAA preferred for body text)

   Priority: high | Effort: medium
   Use icons, patterns, or text labels alongside color-coded information
   Implementation:
     • Add icons to success/warning/error messages

     • Include text labels for status indicators
     • Add underlines to links (not just color)
     • Use shapes in addition to colors for data visualization

   Priority: high | Effort: medium
   Use proper HTML5 elements for better accessibility and SEO
   Implementation:
     • Use <header>, <nav>, <main>, <article>, <aside>, <footer>
     • Implement proper heading hierarchy (h1 → h6, no skipping)
     • Use <button> for clickable actions, <a> for navigation
     • Add ARIA landmarks for complex layouts
     • Use <label> elements for all form inputs
     • Implement proper list structures (<ul>, <ol>) for grouped content

   Priority: high | Effort: medium
   Ensure all interactive elements are easy to tap on mobile devices
   Implementation:
     • Minimum 44x44px (iOS) or 48x48dp (Android) touch targets

     • Increase tap area beyond visible button size if needed
     • Test with various finger sizes and dexterity levels
     • Avoid placing critical actions at screen edges

   Priority: high | Effort: medium
   Implement comprehensive accessible form patterns
   Implementation:
     • Associate <label> with every form input (for/id matching)
     • Use placeholder text as hints, not labels
     • Add aria-required="true" for required fields
     • Show inline validation with both visual and text feedback
     • Group related inputs with <fieldset> and <legend>
     • Display error messages near relevant fields
     • Use autocomplete attributes for common fields
     • Announce validation errors to screen readers (aria-live)

   Priority: high | Effort: medium
   Provide alternative experiences for users who prefer reduced motion

     • Detect prefers-reduced-motion media query
     • Disable or reduce non-essential animations
     • Replace animations with instant transitions or fades
     • Maintain functionality without relying on animations
     • Avoid auto-playing videos or carousels
     • Provide play/pause controls for all motion content
